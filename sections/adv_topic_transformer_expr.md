# Expressiveness of Transformers

## Outline

As in the MLP and CNN section, the basic expressiveness question starts from a ground-truth family and a parametric model family $\{f_\theta:\theta\in\Theta\}$. For transformers, the ground truth is usually not a function of a fixed number of arguments, but a family of sequence-to-sequence computations. Describing such a ground-truth class therefore naturally leads to the theory of computation. Depending on the assumptions on sequence length, positional information, masking, numerical precision, and whether inference is a single forward pass or an iterative generation process with intermediate tokens such as chain of thought, the relevant function classes can range from low-depth circuit classes such as $\mathsf{AC}^0$ and $\mathsf{TC}^0$ to polynomial-time computation and, in more idealized settings, Turing completeness.


1. Model assumptions and resource regime

   We first specify the computational regime. The most important distinction is between sequence length and model size, together with the form of inference: one-pass prediction or iterative generation, with or without chain of thought, prompting, or more agentic interaction. A second distinction is architectural: encoder versus decoder, encoder-decoder versus decoder-only, and transformer versus recurrent or other sequential models. A third distinction concerns structural assumptions such as positional information, masking, and numerical precision. We will also explain the relevant complexity classes and roughly which assumptions lead to classes ranging from $\mathsf{AC}^0$ and $\mathsf{TC}^0$ to polynomial-time computation and, in more idealized settings, Turing completeness.

2. No-CoT transformers as bounded parallel computation

   A transformer that reads the input and immediately answers with one bounded-depth forward pass is best understood as a parallel computation. This is the "no chain-of-thought as a bounded circuit" viewpoint. Under finite-precision or hard-attention assumptions, many transformer variants can be simulated by low-depth circuit or logic classes such as $\mathsf{AC}^0$, $\mathsf{TC}^0$, or first-order logic with counting or majority quantifiers. Such results explain why parity, exact counting, graph connectivity, and some formal languages become separator tasks.

3. Chain of thought as a general computation

   We distinguish parameter expressiveness and prompt expressiveness: the first corresponds more to learning, while the second corresponds more to execution. CoT changes the resource model because the model can write intermediate tokens and then attend to them, so the transformer is no longer only a bounded parallel computation. With trainable parameters, CoT can simulate much more general sequential machines; in the formal literature this is what leads from circuit-style expressiveness toward polynomial-time computation, and in more idealized settings toward Turing completeness. With a frozen pretrained model, CoT prompting is better viewed as steering or eliciting computations that the backbone can already implement, unless the prompt is allowed to be a powerful learned prefix in a theoretical construction.

4. Case studies

   We study case studies separately in the non-CoT and CoT settings.

   * non-CoT

        Formal languages, logic, and counting give the cleanest discrete theory. Results range from limitations on periodic languages and Dyck languages, to exact characterizations of restricted masked hard-attention transformers as star-free languages, to broader logic characterizations for finite- and log-precision models. Counting deserves special attention because many lower bounds and benchmark failures reduce to the difficulty of maintaining exact global counts.

        Associative recall and mechanistic circuits ask whether the model can bind keys to values in context and retrieve the value associated with a query key. This connects formal expressiveness, long-context retrieval, in-context learning, and mechanistic interpretability. Induction heads are one concrete circuit for copy-and-continue behavior; associative-memory analyses study how attention and MLPs can store and retrieve facts.

        Long context raises an expressiveness question because the model must preserve useful positional information as the sequence becomes long, so failures of positional encoding or attention scaling can already break simple retrieval; Needle-in-a-haystack (NIAH) is the standard basic probe for this. A separate but related issue is length generalization: a model trained on shorter sequences may be expressive enough in principle yet still fail on longer ones because it learned a shortcut tied to the training length rather than the underlying algorithm.

   * CoT

        We revisit the same kinds of case studies when intermediate tokens are allowed, and ask how the additional sequential workspace changes the complexity class and the practical behavior.

5. Alternative architectures

   We also compare alternative architectures and alternative ways of carrying state, because explicit CoT is not the only way to add inference-time computation.

   One direction stays close to transformers themselves: looped transformers and architectures that rely more heavily on persistent hidden states.

   A second direction compares transformers with recurrent models, including classical RNN-style systems and newer variants such as Mamba, RWKV, TTT and etc., as well as viewpoints in which transformers can be interpreted as RNN-like systems.

   A third direction adds external or auxiliary computation, such as transformers with memory, and RAG.

   The common theme is that these models use hidden state, recurrence, or memory as alternatives to explicit CoT.

6. In-context learning and prompt-space expressiveness

   In-context learning treats the context as a training set and asks whether a frozen transformer can map examples to a prediction without weight updates. Prompt tuning and prefix tuning ask a related but distinct question: how much can a frozen backbone be controlled by learnable or textual prompt tokens? The literature contains both positive universality constructions and negative limitations; the assumptions about the frozen backbone and allowed prompt length are decisive.

## 1. Model Assumptions and Resource Regime

A transformer is not expressively classified by architecture name alone; the right object is a family of maps indexed by the resource regime. For a fixed length $n$, we can view it either as a sequence-to-sequence map $f_{\theta,n}:\mathcal X^n\to\mathcal Y^{m(n)}$ or as a recognizer $r_{\theta,n}:\Sigma^n\to\{0,1\}$, where the latter induces a language $L_n=\{x:r_{\theta,n}(x)=1\}$. The recognizer viewpoint is the one that connects cleanly to circuit complexity and formal languages, while the seq2seq viewpoint is the one that matches translation, parsing, retrieval, and generation. A Boolean circuit family is the asymptotic analogue: one circuit $C_n$ per input length $n$. Uniformity means there is an efficient Turing machine generating $C_n$ from $1^n$; nonuniformity means there need not be any such generator, so a distinct parameter setting may hide length-specific advice. In the same language, $\mathsf{AC}^0$ means constant depth, polynomial size, and unbounded-fan-in AND/OR/NOT gates, while $\mathsf{TC}^0$ means constant depth, polynomial size, and threshold or majority gates. A Turing machine is the standard sequential model of computation: it has a finite control, a read-write tape, and a head that reads one cell, updates it, and moves left or right. $P$ is the class of languages decidable in polynomial time by a Turing machine, and $NP$ is the class whose yes-instances have polynomial-size certificates verifiable in polynomial time. A language is $P$-complete if it lies in $P$ and every problem in $P$ logspace-reduces to it, so $P$-complete problems are the standard boundary cases when one wants to show that a model has reached full polynomial-time power. Turing completeness means that, under the assumptions of the theorem, the model can simulate any Turing machine.

The standard attention layer makes the resource assumptions visible. With hidden states $X\in\mathbb{R}^{n\times d}$, one head computes $Q=XW_Q$, $K=XW_K$, $V=XW_V$, and then $\mathrm{Attn}(X)=\mathrm{softmax}\!\left((QK^\top)/\sqrt{d_k}+M\right)V$, where the mask $M$ encodes which positions can interact. Masking is not a cosmetic detail: causal masks make decoder-only models autoregressive, bidirectional masks make encoder-style models symmetric over visible tokens, and stricter masks can collapse the interaction pattern to a much weaker local or unique-choice computation. Positional encoding is equally structural. Absolute encodings add a position vector to each token, relative encodings bias pairwise distances, and rotary encodings inject position through phase rotation; without some positional signal, a self-attention encoder is permutation-equivariant, so order-sensitive functions are not directly available. The original encoder-decoder Transformer of [Vaswani et al. 2017](https://arxiv.org/abs/1706.03762) is the baseline reference for this taxonomy, because it already separates source encoding, target decoding, and causal masking.

These assumptions matter because they separate three different senses of "more expressive." First, sequence length versus model size: a fixed-size model running on arbitrary $n$ is a genuine algorithmic family, while a theorem that lets parameters depend on $n$ is much closer to a nonuniform circuit family. Second, precision: finite-precision models can only carry a bounded amount of information per scalar, whereas exact real arithmetic or unbounded precision can encode much more state in a single number. Third, inference protocol: one-pass prediction, autoregressive generation, and explicit scratchpad-style iteration are different resource models. The last distinction is the bridge from bounded parallel computation toward $P$-like sequential computation, and in idealized settings toward Turing completeness. In short, if the model remains one-pass, constant-depth, and finite-precision, the right complexity lens is usually a circuit class; if it is allowed to iterate and externalize state, the right lens shifts toward polynomial-time or Turing-machine simulation.

At fixed sequence length, the canonical positive theorem is [Yun, Bhojanapalli, Rawat, Reddi, and Kumar 2020](https://openreview.net/forum?id=ByxRM0Ntvr), which proves that transformers are universal approximators of continuous sequence-to-sequence functions on compact domains, and that positional encodings remove the permutation-equivariance restriction so arbitrary continuous seq2seq maps can be approximated. The proof strategy is constructive and modular: self-attention first builds a contextual representation that aggregates information across positions, and then position-wise feed-forward layers approximate the desired continuous map on that representation by ordinary universal approximation arguments. This theorem is important, but only in the fixed-length, density-theoretic sense. It does not by itself say that one bounded-size transformer family solves arbitrary-length problems, and it does not give a complexity-theoretic guarantee of efficient computation. A useful cautionary companion is [Luo, Li, Zheng, Liu, Wang, and He 2022](https://openreview.net/forum?id=NQFFNdsOGD), which shows that relative positional encoding does not automatically rescue universality: even with depth and width allowed to grow, some continuous sequence-to-sequence functions remain unapproximable under their assumptions. Other results in this fixed-length approximation family include [Alberti, Dern, Thesing, and Kutyniok 2023](https://proceedings.mlr.press/v221/alberti23a.html), [Takakura and Suzuki 2023](https://proceedings.mlr.press/v202/takakura23a.html), and the prompt-based universality results of [Petrov, Torr, and Bibi 2024](https://proceedings.mlr.press/v235/petrov24a.html) and [Wang, Chauhan, Wang, and Hsieh 2023](https://openreview.net/forum?id=zWxKYyW9ik).

At the opposite end, the bounded-circuit regime gives the cleanest asymptotic upper bounds for constant-depth transformers. A representative theorem is [Merrill, Sabharwal, and Smith 2022](https://aclanthology.org/2022.tacl-1.49/): saturated transformers with floating-point values can be simulated by constant-depth threshold circuits, so their formal-language power is upper-bounded by $\mathsf{TC}^0$-style computation. The proof idea is to compile each layer into a threshold-circuit description: saturated attention behaves like a controlled selection or comparison mechanism, bounded-precision arithmetic can be expressed by threshold predicates, and a constant number of layers composes to a constant-depth circuit family. A closely related result is [Hao, Angluin, and Frank 2022](https://aclanthology.org/2022.tacl-1.46/), which places unique-hard-attention and generalized unique-hard-attention transformers in $\mathsf{AC}^0$, while showing that averaging hard attention can recognize some non-$\mathsf{AC}^0$ languages such as $\mathsf{MAJORITY}$ and $\mathsf{DYCK}$-1 in their setting. Logic characterizations sharpen the same story: [Chiang, Cholak, and Pillay 2023](https://proceedings.mlr.press/v202/chiang23a.html) identify a first-order logic with counting quantifiers that is both an upper and lower bound for fixed-precision transformer encoders, and [Merrill and Sabharwal 2023](https://openreview.net/forum?id=uR8TtWCIsr) show that log-precision transformers admit a first-order characterization with majority quantifiers. The common proof pattern is always the same: bounded precision and bounded depth prevent the model from storing arbitrarily rich state, so attention becomes a form of constant-round counting or thresholding rather than a full sequential algorithm. [Hahn 2020](https://aclanthology.org/2020.tacl-1.11/) is the early limitation theorem that makes this intuition concrete by showing that self-attention cannot model periodic finite-state languages or hierarchical structure unless the number of layers or heads grows with input length.

To move from bounded circuits toward $P$-like or Turing-complete behavior, the decisive change is not just "more parameters" but "more rounds of computation" or "more power in the state representation." A transformer that is allowed to generate a visible scratchpad, or to loop over its own hidden state, is no longer a one-shot circuit; it has become an iterated computation with externalized workspace. In the idealized exact-arithmetic setting, this can go all the way to Turing completeness: [Perez, Barcelo, and Marinkovic 2021](https://www.jmlr.org/papers/v22/20-302.html) prove that hard-attention transformers are Turing complete, and [Bhattamishra, Patel, and Goyal 2020](https://aclanthology.org/2020.conll-1.37/) give a simpler proof and show that even transformers with positional masking but no positional encoding can be Turing complete under their assumptions. The proof strategy is to encode the machine configuration in dense vectors or position-indexed slots, then use attention, masking, and residual updates to simulate one Turing-machine step per layer or decoding step. This is an exact simulation claim, not a practical claim: it depends on idealized arithmetic, carefully chosen encodings, and unbounded time or length. The point for this section is that the same architecture can live in a circuit regime or a Turing-machine regime depending on how much sequential state and numerical idealization we grant it.

Prompting and prefixing define another axis that is easy to miss if one only talks about learned weights. Parameter expressiveness asks how the function class changes as $\theta$ varies; prompt expressiveness asks how the function class changes as a prompt or prefix $p$ varies while $\theta$ is frozen. Formally, one studies $f_{\theta,p}$ rather than $f_\theta$, so the prompt acts like an additional inference-time control surface rather than a new set of learned parameters. This is not the same as ordinary fine-tuning, and it is not the same as chain of thought or other explicit scratchpad schemes, though the resource logic is similar. The sample theorem here is [Petrov, Torr, and Bibi 2024](https://proceedings.mlr.press/v235/petrov24a.html): prefix-tuning a pretrained transformer can be a universal approximator of sequence-to-sequence functions, and in their construction a single attention head already suffices to approximate any continuous function. The proof idea is that the prefix can steer attention into a rich family of basis features, so the frozen backbone plus prefix behaves like a programmable approximator. [Wang, Chauhan, Wang, and Hsieh 2023](https://openreview.net/forum?id=zWxKYyW9ik) complement this with both positive and negative results for soft-prompt tuning: prompt tuning can be universal in stylized settings, but finite-depth fixed-weight models also have hard limitations and lower bounds on the prompt budget. The broader lesson is that inference-time expressiveness is a separate resource from parameter expressiveness; a frozen model with a long or learnable prefix can implement behaviors unavailable to the same model with no prompt, but this is still an assumption about how the model is being used, not a new theorem about the backbone alone.

The regime map is therefore the main content of this section. Fixed-length, continuous seq2seq approximation lives in the universality literature; bounded precision and constant depth push transformers into $\mathsf{AC}^0$, $\mathsf{TC}^0$, or first-order logic with counting or majority; iterated generation and explicit scratchpads shift the relevant notion of computation toward $P$ or, under idealized encodings, Turing completeness; and prompt or prefix tuning moves part of the expressive burden from learned parameters into inference-time control. The rest of the chapter will repeatedly reuse this map, but the map itself is the critical first step: before asking what transformers can express, we have to say exactly which transformer, with which mask, which positional signal, which precision, which length regime, and which inference protocol.

## 2. No-CoT Transformers as Bounded Parallel Computation

A no-CoT transformer is best understood as a fixed-depth parallel machine. The model reads the whole input, performs a bounded number of synchronous attention-and-MLP rounds, and emits an answer immediately, with no visible scratchpad and no length-dependent loop. That makes the right abstractions circuit-theoretic and logical rather than sequential: a circuit family is a sequence $\{C_n\}_{n\ge 1}$ with one circuit per input length, bounded depth means $\sup_n \mathrm{depth}(C_n) < \infty$, and uniformity means a small algorithm can generate $C_n$ from $n$ rather than hiding arbitrary advice in the weights. $\mathsf{AC}^0$ is the class of constant-depth, polynomial-size, unbounded-fan-in AND/OR/NOT circuits; $\mathsf{TC}^0$ replaces AND/OR by threshold or majority gates. On strings, a formal-language recognizer is just a yes/no family that accepts $x\in\Sigma^n$ uniformly over all $n$, usually via a classification token or designated output position. The logical side uses the same bounded-parallel idea in another language: first-order logic $\mathsf{FO}[<]$ talks about positions $1,\dots,n$ with order and letter predicates, counting quantifiers $\exists^{\ge k}x\,\varphi(x)$ can say how many positions satisfy a property, and majority quantifiers $\mathsf{Maj}_x\,\varphi(x)$ ask whether more than half of the positions do. Finite precision matters because without a bit-width bound, a single real coordinate can hide unbounded information; under the standard discrete assumptions, hidden states and scores carry only $O(1)$ or $O(\log n)$ effective bits, so the forward pass can be discretized. Hard attention means one head selects a single maximizing position via $\arg\max$, soft attention uses a softmax-weighted average, and average-hard or saturated attention are intermediate variants that are still much easier to analyze than fully continuous softmax. CoT is the contrasting regime, because the model can write and reread its own intermediate tokens, which adds visible sequential workspace and breaks the bounded-parallel picture.

### Limitations

The cleanest limitation results say that a fixed-depth transformer cannot robustly express certain counting or nesting behaviors unless some architectural resource grows with input length. [Hahn 2020](https://direct.mit.edu/tacl/article/doi/10.1162/tacl_a_00306/43545/Theoretical-Limitations-of-Self-Attention-in) is the canonical early theorem here: across both soft and hard attention, fixed-size self-attention cannot model periodic finite-state languages or hierarchical structure unless the number of layers or heads scales with the sequence length. The right way to read this is as a bounded-depth phenomenon, not a mysterious NLP failure. A fixed number of synchronous rounds can only propagate and refine a finite amount of global phase information, so long-range periodicity and unbounded bracket nesting eventually become indistinguishable. The proof strategy is an invariance or indistinguishability argument over long strings, where the fixed architecture forces many long inputs into the same coarse interaction pattern, so the model cannot consistently separate the relevant counts or nesting depths.

Other major results in this category include [Formal Language Recognition by Hard Attention Transformers: Perspectives from Circuit Complexity](https://aclanthology.org/2022.tacl-1.46/), which shows that UHAT and GUHAT lie in $\mathsf{AC}^0$, so classic separators such as parity and Dyck-style languages are out of reach in that setting; [Logical Languages Accepted by Transformer Encoders with Hard Attention](https://proceedings.iclr.cc/paper_files/paper/2024/hash/5f0fdc1acd47431f7f3bb8ee85598cef-Abstract-Conference.html), which strengthens the picture by exhibiting an $\mathsf{AC}^0$ language not recognized by UHAT; and the classical constant-depth lower-bound line that begins with parity and majority. In tutorial terms, the message is that no-CoT transformers are not generic sequential devices in disguise. Once the architecture is frozen, the model can still be powerful, but it inherits the same kinds of separator tasks that have long organized circuit complexity.

### Circuit Upper Bounds

The positive side of the story is that many transformer variants compile cleanly into constant-depth circuits. A representative theorem is [Saturated Transformers are Constant-Depth Threshold Circuits](https://aclanthology.org/2022.tacl-1.49/): Merrill, Sabharwal, and Smith show that saturated transformers with floating-point values are simulable by constant-depth threshold circuits, so the natural upper bound is $\mathsf{TC}^0$. The proof works layer by layer. Each attention head is reduced to a bounded comparison and routing gadget, the weighted sums are implemented by shallow threshold-style arithmetic, and the fixed number of layers means the entire computation stays at constant depth. Conceptually, this is the point where the architecture has moved beyond pure $\mathsf{AC}^0$ but is still firmly inside bounded parallelism. The result is especially useful because it captures a more practical attention regime than exact hard attention, while still giving a rigorous classical upper bound.

Other major results in this category include [Formal Language Recognition by Hard Attention Transformers: Perspectives from Circuit Complexity](https://aclanthology.org/2022.tacl-1.46/), which places UHAT and GUHAT in $\mathsf{AC}^0$ and shows that AHAT can recognize MAJORITY and DYCK-1; [Tighter Bounds on the Expressivity of Transformer Encoders](https://proceedings.mlr.press/v202/chiang23a.html), which ties fixed-precision encoders to a counting-logic fragment and to a constant-depth upper-bound regime; and [Logical Languages Accepted by Transformer Encoders with Hard Attention](https://proceedings.iclr.cc/paper_files/paper/2024/hash/5f0fdc1acd47431f7f3bb8ee85598cef-Abstract-Conference.html), which shows that UHAT and AHAT can recognize richer logical fragments once unary numerical predicates and counting terms are allowed. The common pattern is that attention is still parallel communication, not iterative search. Once the precision is bounded, the entire forward pass can be re-expressed as a shallow circuit family.

### Logic Characterizations

Logic is the cleanest language for saying what these bounded parallel computations can do. [Transformers Implement First-Order Logic with Majority Quantifiers](https://openreview.net/forum?id=W668diqwp4l) shows that any transformer neural network can be translated into a first-order sentence with majority quantifiers, and [A Logic for Expressing Log-Precision Transformers](https://proceedings.neurips.cc/paper_files/paper/2023/hash/a48e5877c7bf86a513950ab23b360498-Abstract-Conference.html) sharpens that picture for $\log n$-precision classifiers. The proof idea is classical and robust: compile the network into a highly uniform threshold circuit, then invoke the circuit-to-logic correspondence that identifies threshold-style computation with majority logic. In that translation, each layer becomes a formula describing which positions can influence which others, how many of those positions satisfy a subformula, and whether a majority of them do. This is why transformer expressiveness results so often come out as fragments of $\mathsf{FO}$ rather than as ad hoc language classes. Counting quantifiers are the natural formal way to talk about global aggregation inside a bounded-depth model, and majority quantifiers are the special case that mirrors threshold gates directly.

Other major results in this category include [Tighter Bounds on the Expressivity of Transformer Encoders](https://proceedings.mlr.press/v202/chiang23a.html), which identifies a variant of first-order logic with counting quantifiers that upper-bounds fixed-precision transformer encoders and lower-bounds general transformer encoders; the same paper's two-sided statement is important because it moves the field from loose upper bounds toward an almost exact logic characterization; and [A Logic for Expressing Log-Precision Transformers](https://proceedings.neurips.cc/paper_files/paper/2023/hash/a48e5877c7bf86a513950ab23b360498-Abstract-Conference.html), which is the tightening step that restores uniform attention while still keeping the computation inside $\mathsf{FO}$ with majority. In plain terms, fixed-precision and log-precision assumptions are not just engineering details, they determine which logical separators, such as counting, parity, and majority, the architecture can or cannot express.

### Exact Characterizations For Restricted Models

The strongest theorems are the exact ones, where the transformer class and the logic class line up on the nose. [Masked Hard-Attention Transformers Recognize Exactly the Star-Free Languages](https://proceedings.neurips.cc/paper_files/paper/2024/hash/13d7f172259b11b230cc5da8768abc5f-Abstract-Conference.html) is the cleanest example. Under strict masking, where each position cannot attend to itself, and with no position embeddings, masked hard-attention transformers are equivalent to linear temporal logic, hence exactly the star-free languages. This is a satisfying result because it turns a design choice into an algebraic classification. The proof uses Boolean RASP as an intermediate language: transformer computations are first rewritten as Boolean sequence programs, then translated into temporal logic, and finally matched with the standard star-free/$\mathsf{FO}[<]$ correspondence. Once that bridge is in place, many familiar regular-language facts become transformer facts for free.

Other major results in this category include [Logical Languages Accepted by Transformer Encoders with Hard Attention](https://proceedings.iclr.cc/paper_files/paper/2024/hash/5f0fdc1acd47431f7f3bb8ee85598cef-Abstract-Conference.html), which shows that UHAT encoders can recognize a rich fragment of $\mathsf{AC}^0$-definable languages, namely those definable in first-order logic with arbitrary unary numerical predicates, while AHAT encoders can handle the same logic even after adding counting terms; [Thinking Like Transformers](https://proceedings.mlr.press/v139/weiss21a.html), which introduced RASP and made this style of intermediate-language proof natural; and the recent transformer-formal-language work showing that strict masking, positional predicates, and depth each increase expressive power in controlled, characterizable ways. The main lesson is that exact characterizations usually appear only after the model is restricted enough to make the combinatorics rigid, but once that rigidity is present the result is much sharper than a mere upper bound.

The overall picture is simple. In the no-CoT regime, a transformer is a uniform bounded-parallel computation, not a sequential program. Under hard attention it often sits near $\mathsf{AC}^0$, under broader discretizable attention it moves toward $\mathsf{TC}^0$, and in the logic mirror it becomes a fragment of first-order logic with counting or majority. Exact characterizations show that the boundary is not vague, it is pinned down by masking, positional structure, and precision. CoT is the separate regime, because once the model can externalize intermediate tokens, the resource being measured is no longer just parallel depth.

## 3. Chain of Thought as a General Computation

Chain of thought (CoT) and scratchpad should be read as a resource model, not just as an explanation format. In the formal literature, a decoder-only transformer does not map an input $x$ directly to an answer $y$; instead it generates a visible intermediate trace $z_1,\dots,z_T$ and only then emits the final output,
$$
x \to z_1 \to z_2 \to \cdots \to z_T \to y.
$$
The key point is that the $z_t$ tokens are sequential workspace. They are append-only, globally visible to later attention heads, and can therefore store state across steps in a way hidden activations alone cannot. In that sense CoT is a computation primitive: it turns one forward pass into $T$ rounds of serial computation over an external tape. A useful separation is between trainable-weight expressiveness and prompt expressiveness. Trainable-weight results ask what a model can implement for some learned $\theta$; prompt-based results fix $\theta$ and vary only the text prefix. Soft prompts and prefix tuning belong on the trainable side because the prefix parameters are learned, while ordinary CoT prompting belongs on the frozen side because it only steers a fixed backbone.

Because the workspace grows with $T$, the natural comparison class is not a bounded-depth circuit class but a time/space complexity class. When $T$ is polynomial in the input length, the right target is $\mathsf{PTIME}$ or $\mathsf{P}$. This is where $P$-complete problems matter: a language is $P$-complete if it belongs to $P$ and every problem in $P$ can be reduced to it by a logspace reduction, so $P$-complete problems are the standard hard boundary cases for full polynomial-time computation. Turing completeness is a stronger and more idealized claim; it belongs to the boundary of the theory, not to the core empirical CoT literature.

### 3.1 Formal CoT Hierarchies

Sample theorem: [Merrill and Sabharwal 2023](https://arxiv.org/abs/2310.07923) show that, under their finite-precision and projected/generalized pre-norm assumptions, CoT induces a genuine hierarchy: logarithmic-length decoding gives only a small extension over the no-CoT regime, linear-length decoding suffices to recognize all regular languages, linear decoding still stays within context-sensitive languages, and polynomial-length decoding characterizes $\mathsf{P}$ exactly. Here a regular language is one recognized by a finite automaton, while a context-sensitive language is one recognized by a linear-bounded automaton, so the theorem is locating CoT inside the classical Chomsky-style hierarchy. Proof idea: they build a layer-norm hash that lets the decoder recover prior state from the generated trace, then simulate one automaton or Turing-machine step per generated token. The converse direction bounds a transformer with $T$ generated tokens by simulating its forward passes on a multitape Turing machine within polynomial time or bounded space. Other major results in this family include the log-step near-no-go bound, the context-sensitive upper bound for linear steps, and the polynomial-step corollaries for standard $P$-complete problems such as Circuit Value and Horn-SAT.

The important conceptual move here is that CoT is not treated as a human-readable explanation; it is treated as an externalized state channel. The intermediate tokens need not be natural language. They can be symbols, tags, or any finite alphabet that the model later attends back to. That is what makes the formal hierarchy work: the model is not merely "thinking out loud," it is using its own output as writable memory.

### 3.2 Serial Computation and Circuit Power

Sample result: [Li et al. 2024](https://arxiv.org/abs/2402.12875) show that constant-depth transformers with constant-bit precision and $O(\log n)$ embeddings can solve any problem computable by Boolean circuits of size $T$ when given $T$ CoT steps, whereas without CoT the same model family is bounded by $\mathsf{AC}^0$ under their precision assumptions. Proof idea: each CoT step functions as one serial stage of computation, with the scratchpad carrying intermediate gate values or other state forward, so the transformer evaluates an inherently serial program rather than trying to compress it into a single parallel pass. Other major results in this paper include the sharper $\mathsf{AC}^0$ upper bound without CoT, and empirical gains on composition of permutation groups, iterated squaring, and Circuit Value Problem tasks.

This result is useful because it isolates the exact resource that CoT adds: not more parameters, but more sequential rounds of computation. That is why the paper frames CoT as a way to overcome the weakness of low-depth transformers on inherently serial problems. The proof is essentially a compilation argument: if a task can be carried out by a circuit of size $T$, then a length-$T$ scratchpad can carry the intermediate state of that circuit one step at a time.

### 3.3 Turing Completeness as a Boundary Case

Sample theorem: [Perez et al. 2021](https://www.jmlr.org/papers/v22/20-302.html) prove that a hard-attention transformer is Turing complete. Proof idea: encode a Turing-machine configuration in dense internal representations, use hard attention to address the relevant tape cell or state, and use feed-forward layers to implement the finite transition table; in other words, attention supplies addressable memory and the dense representation supplies the tape alphabet. Other major results of this kind are related completeness constructions for recurrent or looped architectures and for models with external memory; these are useful as idealized boundary points, but they are not the same thing as CoT prompting on a frozen language model.

This completeness literature is relevant to CoT only as an upper boundary. It says that under strong idealizations, a transformer-like architecture can simulate arbitrary computation. It does not say that ordinary prompting yields that power, and it does not say that a finite pretrained model will learn to exploit it. In a survey of CoT as general computation, it is best treated as the extreme end of the same story: once a model has enough exact state, exact addressing, and enough steps, the architecture is no longer confined to bounded parallel reasoning.

### 3.4 Frozen-Prompt CoT and Empirical Reasoning

Sample result: [Wei et al. 2022](https://arxiv.org/abs/2201.11903), [Kojima et al. 2022](https://arxiv.org/abs/2205.11916), and [Wang et al. 2022](https://arxiv.org/abs/2203.11171) show that few-shot CoT, zero-shot "Let's think step by step," and self-consistency sampling can substantially improve benchmark accuracy on arithmetic and symbolic reasoning tasks. Proof idea: there is no complexity-theoretic proof here; the evidence is benchmark ablation. The mechanism is empirical and distributional: the prompt elicits a latent reasoning mode already present in the backbone, and self-consistency reduces decoding noise by sampling multiple traces and voting over the final answer. Other major empirical results include scratchpad training [Nye et al. 2021](https://arxiv.org/abs/2112.00114), least-to-most prompting, program-of-thought prompting, ReAct-style reasoning-and-action traces, and many follow-on self-verification or answer-calibration methods.

These papers matter, but they should not be read as architectural expressiveness theorems. They study behavior of a fixed or fine-tuned model under a decoding protocol, not the formal language class of the backbone itself. Scratchpad training sits on the trainable side of the line because the model is optimized to emit intermediate computation steps; zero-shot and few-shot CoT prompting sit on the frozen side because the model weights are unchanged and only the prompt is modified. That distinction is the core taxonomy: trainable-weight results change the function class, prompt-based results change the execution context, and empirical reasoning papers measure whether that change improves benchmark performance.

The clean takeaway is that CoT is best understood as sequential workspace. In formal theory, extra workspace changes the computational class, sometimes all the way up to $\mathsf{P}$; in prompting papers, the same idea is used as an execution-time control signal for a frozen model; in empirical reasoning papers, the question is whether that control signal improves benchmark accuracy. Those are related questions, but they answer different parts of the expressiveness problem.

## 4. Case Studies

The case studies are easiest to read if they are split by inference regime. In the non-CoT setting, the transformer must answer in one bounded pass, so the natural questions are about finite-state recognition, exact counting, content-addressed retrieval, and length extrapolation. In the CoT setting, the model is allowed to externalize intermediate state into generated tokens, so the natural questions become algorithmic composition, iterative update, and whether scratchpad tokens change the reachable complexity class.

### 4.1 Non-CoT case studies

#### Formal languages: regular, star-free, and Dyck

This is the cleanest discrete family because the target classes are standard from automata theory. Regular languages are those recognized by finite automata. Star-free regular languages are the aperiodic regular languages, equivalently the languages definable in first-order logic over words or in linear temporal logic without Kleene star. Dyck-$k$ is the canonical context-free family of balanced bracket languages, where "context-free" means recognized by a pushdown automaton rather than by a finite automaton. This family tests whether the model can maintain stack-like nesting rather than just finite-state memory.

A representative exact result is [Yang, Chiang, and Angluin 2024](https://proceedings.neurips.cc/paper_files/paper/2024/hash/13d7f172259b11b230cc5da8768abc5f-Abstract-Conference.html), which proves that masked hard-attention transformers with strict masking and no position embeddings recognize exactly the star-free languages. The proof strategy is conceptually simple and is exactly the kind of proof that belongs in a survey like this: the authors introduce Boolean RASP as an intermediate programming language, show an equivalence between Boolean RASP and the transformer variant under study, and then transfer the known correspondence between linear temporal logic and star-free languages. The point is not just an upper bound; it is an exact characterization that shows how attention direction, masking, and positional information control formal-language power.

The rest of the family is just as instructive. [Hao, Angluin, and Frank 2022](https://aclanthology.org/2022.tacl-1.46/) show that unique hard-attention variants lie in $\mathsf{AC}^0$, while averaging hard attention can recognize languages such as $\mathsf{MAJORITY}$ and $\mathsf{DYCK}$-1. Earlier, [Bhattamishra, Ahuja, and Goyal 2020](https://aclanthology.org/2020.emnlp-main.576/) gave explicit constructions for a subclass of counter languages, including Dyck-1 and related nested languages, and [Weiss, Goldberg, and Yahav 2021](https://proceedings.mlr.press/v139/weiss21a.html) compiled Dyck-language and histogram solutions into the RASP framework, which is the key bridge between symbolic sequence programs and transformer implementations. The broader survey point is that regular languages, star-free regular languages, and Dyck languages are not interchangeable benchmarks; they probe different memory resources, and the literature has gradually separated them into distinct expressiveness regimes.

#### Counting

Counting is the next simplest family, but it is less trivial than it looks because exact multiplicity is a global property. The canonical tasks are histograms, majority, parity, and related frequency-estimation problems. A model that can tell whether a token occurs at least once is not necessarily able to count how many times it occurs, and a model that can count up to a fixed bound is not necessarily able to generalize to longer contexts.

A representative result is [Behrens, Biggio, and Zdeborova 2025](https://proceedings.mlr.press/v267/behrens25a.html), which analyzes the histogram task in small transformers and identifies two distinct counting regimes, relation-based counting and inventory-based counting. Their experimental logic is useful as a template for the whole family: instead of only reporting accuracy, they vary vocabulary size, embedding dimension, token mixing, feed-forward capacity, and simple architectural details such as softmax and beginning-of-sequence tokens, then inspect which counting strategy the trained model settles on. The main lesson is that counting is not a single capability. It can be implemented either by directly maintaining relational statistics over tokens or by building an explicit inventory-like memory, and those two solutions place different burdens on attention and feed-forward layers.

Other results in this family reinforce the same theme. [Brody, Alon, and Yahav 2023](https://aclanthology.org/2023.findings-acl.895/) show that LayerNorm is not merely a training stabilizer, because its geometry matters for functions like majority. Counting also appears as a separator in regular-language work, since parity and majority compress a whole sequence into a tiny global summary, and finite-precision circuit-style analyses often use exactly these tasks to separate $\mathsf{AC}^0$, $\mathsf{TC}^0$, and stronger counting regimes.

#### Associative recall

Associative recall asks whether the model can bind a key to a value in context and later recover the matching value when queried with the key. The simplest form is a list of key-value pairs followed by a query key, with the correct output being the associated value. This family matters because it is the smallest synthetic version of factual recall, entity binding, and in-context lookup.

A representative result is [Nichani, Lee, and Bietti 2024](https://openreview.net/forum?id=PtYojIoW0u), which shows that a shallow transformer with one self-attention layer followed by an MLP can achieve 100 percent accuracy on a synthetic factual recall task whenever either the self-attention parameters or the MLP parameters scale essentially linearly with the number of facts. The proof strategy is instructive: the model is analyzed as an associative memory, and the result shows a tradeoff between storing facts in value matrices and storing them in the MLP. In other words, the transformer does not need a single monolithic memory mechanism; it can distribute storage across different submodules and still reach near-optimal capacity.

The surrounding literature makes the mechanism even more concrete. [Olsson et al. 2022](https://transformer-circuits.pub/2022/in-context-learning-and-induction-heads/index.html) identify induction heads as a circuit for copy-and-continue behavior, which is a close mechanistic cousin of associative recall. [Feng and Steinhardt 2024](https://openreview.net/forum?id=zb3b6oKO77) show that language models represent binding information through binding-ID vectors in context. On the associative-memory side, [Hu et al. 2024](https://proceedings.mlr.press/v235/hu24a.html), [Santos et al. 2024](https://proceedings.mlr.press/v235/santos24a.html), and [Smart, Bietti, and Sengupta 2025](https://proceedings.mlr.press/v267/smart25a.html) connect attention to modern Hopfield or dense associative memory models, which makes the retrieval family one of the most mechanistically grounded parts of the transformer expressiveness literature.

#### Long-context retrieval and NIAH

Long-context retrieval is the retrieval family once the sequence is long enough that positional encoding, distractor suppression, and attention scaling become part of the task. The simplest benchmark is NIAH, needle in a haystack: one relevant item is hidden in a long distractor prompt, and the model must recover it. That probe is useful, but by itself it is too weak to characterize long-context capability.

A representative result is [Hsieh et al. 2024](https://github.com/NVIDIA/RULER), which introduces RULER specifically to go beyond vanilla NIAH. Their benchmark expands the needle family to multiple needle types and quantities and adds multi-hop tracing and aggregation tasks. The core empirical logic is sharp: many models score almost perfectly on vanilla NIAH, yet performance drops substantially as context length and task complexity increase. So NIAH is necessary as a sanity check, but it is not sufficient as an evaluation of real long-context understanding.

The broader long-context retrieval literature pushes the same point. [Bertsch et al. 2024](https://arxiv.org/abs/2405.00200) use many-shot in-context learning as a testbed and show that long-context gains often come more from attending back to similar examples than from cumulative task learning. [Su et al. 2024](https://arxiv.org/abs/2407.12883) introduce BRIGHT, a reasoning-intensive retrieval benchmark, and show that even when the relevant document is present, retrieval can remain hard if relevance requires reasoning rather than lexical overlap. [Bai et al. 2023](https://arxiv.org/abs/2308.14508) provide LongBench, which broadens the evaluation to bilingual, multitask long-context understanding. The survey lesson is that NIAH is the floor, not the ceiling; real long-context retrieval involves distractor robustness, multi-hop selection, and aggregation over the retrieved evidence.

#### Length generalization

Length generalization asks whether a model trained on shorter sequences still behaves algorithmically on longer ones. This is not the same as ordinary accuracy, because a model can fit short examples perfectly while still learning a shortcut that breaks as soon as the input length changes. The relevant question is therefore whether the model learned the underlying algorithm or only a length-specific heuristic.

A representative result is [Zhou et al. 2024](https://proceedings.iclr.cc/paper_files/paper/2024/hash/45ed1a72597594c097152ef9cc187762-Abstract-Conference.html), which frames length generalization through RASP-L. Their main claim is that tasks with short RASP-L descriptions tend to length generalize, and that scratchpad design matters: a good scratchpad can expose the right algorithm, while a poor one can make generalization worse. The proof logic is a mix of compilation and experimentation. They compare a target task to its short symbolic description, train transformers from scratch, and then test whether the learned behavior extrapolates to longer inputs.

The rest of the family includes [Weiss et al. 2021](https://proceedings.mlr.press/v139/weiss21a.html), who make the RASP compilation viewpoint explicit, and more recent theoretical work such as [Trauger and Tewari 2024](https://proceedings.mlr.press/v238/trauger24a.html) on sequence-length-independent generalization bounds and [Chen, Ma, and Li 2025](https://proceedings.mlr.press/v267/chen25ar.html) on non-asymptotic length complexity. The most important classification point is that length generalization is a property of the learned solution, not just the architecture. A model can be expressive enough in principle and still fail to extrapolate if the training distribution does not expose the right algorithmic structure.

### 4.2 CoT case studies

Once chain of thought is allowed, the case studies change character. The question is no longer only whether the model can compute a function in one pass, but whether generating intermediate tokens gives it enough sequential workspace to simulate a more general algorithm. This is why CoT case studies are naturally drawn from repeated composition, arithmetic derivation, dynamic programming, and classical complexity separators.

A representative result is [Feng et al. 2023](https://proceedings.neurips.cc/paper_files/paper/2023/hash/dfc310e81992d2e4cedc09ac47eff13e-Abstract-Conference.html). Their theorem has two parts. First, bounded-depth transformers cannot directly solve even basic arithmetic and equation-style reasoning tasks unless the model size grows super-polynomially with the input length. Second, autoregressive transformers with a suitable math-language format can solve those same tasks by generating chain-of-thought derivations step by step. The proof strategy is exactly what one would want in a tutorial survey: a lower bound for the direct-answer regime, then a constructive upper bound for the scratchpad regime. They also show that the same idea extends to a broad class of dynamic-programming problems, which is why CoT is usually associated with reasoning tasks that have an iterative substructure.

The broader theory paper is [Merrill and Sabharwal 2023](https://arxiv.org/abs/2310.07923), which gives a complexity hierarchy for CoT itself. In their setting, logarithmic CoT gives only a modest extension, linear CoT is enough to recognize all regular languages, and polynomial CoT reaches polynomial-time computation. That result is important because it clarifies what CoT is doing in the theory: it is not just making the prompt longer, it is changing the computational resource model by letting the network write and reread its own intermediate state.

Other canonical separators in the CoT family are best understood as repeated-composition tasks. [Li et al. 2024](https://proceedings.iclr.cc/paper_files/paper/2024/hash/3309b4112c9f04a993f2bbdd0274bba1-Abstract-Conference.html) make the serial-computation intuition explicit by showing that a transformer with $T$ CoT steps can solve problems computable by Boolean circuits of size $T$ under their construction. Permutation composition is the cleanest synthetic benchmark in this direction, because the model must keep updating an internal state as it reads a sequence of permutations; [Li, Guo, Andreas, and coauthors 2025](https://proceedings.mlr.press/v267/li25r.html) study exactly this kind of state tracking, and [Wang et al. 2025](https://proceedings.mlr.press/v291/wang25a.html) study $k$-fold composition as a learnability problem. As an inference from this literature, iterated squaring is best read as the same repeated-update pattern in a different algebraic guise, while circuit value is the classical complexity-theoretic form of the same separator. The benchmark label changes, but the structural test does not: can the model carry a latent object through many update steps without collapsing the computation into a shallow shortcut?

A useful practical takeaway is that CoT case studies are not just about better answers. They are about a different execution model. When a model writes intermediate tokens, it is no longer limited to one bounded parallel pass, so problems that looked like expressive dead ends in the no-CoT regime can become tractable. The strongest survey picture is therefore not "CoT makes transformers smarter" in a vague sense, but "CoT gives transformers extra sequential workspace, and that workspace reclassifies which families are easy, hard, or separator tasks."

## 5. Alternative Architectures

The common theme in this section is that the model gets extra computation without writing a visible chain of thought. The scratchpad is moved into latent state, repeated depth, a separate memory, or fast weights. Formally, the update is no longer just a single feed-forward map from input to output; it is a state transition such as $h_{t+1}=F_\theta(h_t,x_t)$, possibly with an auxiliary memory update as well. The expressiveness question is therefore: what computations can be realized by the state update itself?

### 5.1 Looping and Depth-Wise Recurrence

A looped transformer is a transformer block whose parameters are reused across multiple iterations. The same block is applied repeatedly to a latent sequence, so depth plays the role of time. This is the transformer analogue of recurrence: the model does not have to write intermediate reasoning into tokens, but it still gets more compute by iterating on hidden activations. The [Universal Transformer](https://openreview.net/forum?id=HyzdRiR9Y7) is the canonical early example, and [Giannou et al. 2023](https://proceedings.mlr.press/v202/giannou23a.html) show that looped transformers can be programmed as universal computers. The proof idea is constructive: one constant-depth block is arranged to implement basic machine operations such as control flow, function calls, and memory read/write, and the loop turns those primitives into an iterative algorithm.

Other major results in this class include recurrent-transformer variants with adaptive halting or depth sharing, such as [Investigating Recurrent Transformers with Dynamic Halt](https://openreview.net/forum?id=JGcDanzlpN), and later work on chunk-wise recurrence. The common mathematical point is that shared parameters across depth can turn a transformer from a bounded parallel computation into an iterative one.

### 5.2 Recurrent Hidden State, Linear Attention, and Selective State Spaces

A recurrent model carries a hidden state forward token by token: $h_{t+1}=F_\theta(h_t,x_t)$. Here the hidden state is the latent workspace, and the expressiveness question is whether that workspace can store enough information to support the target computation. The classical upper-bound result is [Siegelmann and Sontag 1995](https://doi.org/10.1006/jcss.1995.1013): finite-size recurrent neural nets with sigmoidal units and rational weights can simulate Turing machines. The proof strategy is to encode the machine configuration into real-valued state and design the recurrent update so that one step of the network matches one step of the simulated machine.

A useful algebraic bridge is [linear attention](https://proceedings.mlr.press/v119/katharopoulos20a.html). Once attention is factorized through a kernel feature map, the causal computation can be rewritten as a running prefix summary, so the transformer becomes recurrent in form rather than only in spirit. Modern recurrent variants such as [RWKV](https://openreview.net/forum?id=7SaXczaBpG) and [Mamba](https://openreview.net/forum?id=tEYskw1VY2) fit the same broad class: RWKV packages attention-like behavior into an explicit recurrence, while Mamba uses an input-selective state-space update that decides what to keep and what to forget. Theoretical analysis is mixed. [Chung and Siegelmann 2021](https://openreview.net/forum?id=IWJ9jvXAoVQ) strengthen the positive side by showing a bounded-precision RNN with a growing memory module can still be Turing complete, while [Merrill, Petty, and Sabharwal 2024](https://openreview.net/forum?id=QZgo9JZpLq) show that state-space models remain limited to low-complexity classes such as $\mathsf{TC}^0$ on the state-tracking problems they study. A more structural account appears in [Cirone et al. 2024](https://openreview.net/forum?id=3SzrqwupUx), which identifies selectivity as the crucial mechanism in modern selective state-space models.

The other results to keep in mind here are the hierarchy papers on recurrent architectures, the formal-language analyses of state-space models, and the growing literature on how much selectivity or multiplicative gating changes the effective expressive power of a recurrent backbone.

### 5.3 External Memory and Retrieval

External memory means that the controller is not the only place where information can live. A writable memory adds a separate store $M_t$ that can be read from and written to; retrieval adds a read-only store, where the model first selects relevant passages and then conditions on them. The representative memory result is [Neural Turing Machines](https://arxiv.org/abs/1410.5401), which pair a controller with differentiable read/write memory. The proof idea is again constructive: content-based and location-based addressing make the memory behave like a differentiable tape, so the controller can implement algorithmic operations such as copy, sort, and associative recall.

The main descendants of this line are [End-to-End Memory Networks](https://arxiv.org/abs/1410.3916), [Hybrid computing using a neural network with dynamic external memory](https://www.nature.com/articles/nature20101), and transformer-native memory interfaces such as [Token Turing Machines](https://openreview.net/forum?id=3m_awcLrg8E). [Retrieval-Augmented Generation](https://arxiv.org/abs/2005.11401) is the read-only version of the same idea: the model computes over a latent document variable $z$ sampled from an external corpus, so the generator's effective hypothesis class is no longer limited to what is stored in the weights alone. The important expressiveness point is that the retriever changes the state space. If the corpus contains the needed fact or subcomputation, the model can condition on it even when the parametric backbone would not have enough internal capacity to memorize it.

### 5.4 Fast Weights and Test-Time Training

Fast-weight models keep the scratchpad inside the parameters of a small inner learner rather than in a token sequence. A Test-Time Training layer makes this idea explicit: the hidden state is itself a model that is updated on the current input stream by an inner optimization rule. In [Learning to (Learn at Test Time): RNNs with Expressive Hidden States](https://openreview.net/forum?id=eifW0W0xgt), the key result is that the hidden state can be treated as a learned predictor, not merely a vector of activations. The theoretical argument is to unroll the inner learning loop: each inference step is also a training step, so the model stores context by changing fast weights rather than by appending more visible tokens.

This line has clear precursors in [Using Fast Weights to Attend to the Recent Past](https://arxiv.org/abs/1610.06258). More recent theory on [Test-Time Training Provably Improves Transformers as In-context Learners](https://openreview.net/forum?id=bma2FB5MNs) and [Test time training enhances in-context learning of nonlinear functions](https://openreview.net/forum?id=sb3H1k3vCG) treats the same basic point from the optimization side: if the model can update an internal learner on the fly, then part of the computation moves from the forward pass into the update rule itself.

### Takeaway

These architectures are not all doing the same thing. Looping gives repeated latent computation with shared parameters. Recurrent hidden state gives a compact state machine. External memory gives a separate store whose contents need not fit in the weights. Fast weights give the model an inner learner that adapts at test time. The useful distinction is therefore not just "transformer or not transformer," but which resource carries the scratchpad and how strong the state-update rule is under the precision and recurrence assumptions of the model.

## 6. In-Context Learning and Prompt-Space Expressiveness

In-context learning (ICL) asks whether a transformer can adapt to a task from examples in its context window without updating any weights. If a prompt is
$$
D=((x_1,y_1),\ldots,(x_k,y_k),x_q),
$$
then the model should output a prediction for $x_q$ that behaves as if it had learned from $D$. A useful formalization is that a transformer implements an in-context learning algorithm $\mathcal A$ when its forward pass realizes the map $D\mapsto \mathcal A(D)$ and the query output agrees with $\mathcal A(D)(x_q)$ on a task family. In the linear-regression literature, $\mathcal A$ is often gradient descent, ridge regression, preconditioned gradient descent, or Newton's method.

Prompt-space expressiveness is the analogous question with the backbone frozen. If $B$ is a pretrained transformer and $p$ is a prompt parameter, then
$$
\mathcal F_{\mathrm{prompt}}(B)=\{x\mapsto B(p,x): p\in\Pi\}
$$
is the function class induced by varying only the prompt. In most theory papers, prompt tuning and soft prompting mean learning continuous prompt embeddings at the input layer; prefix tuning is the stronger interface where learned vectors are injected as extra key/value states, often layer by layer, rather than only at the input. This is different from trainable-weight expressiveness, where all parameters may move, and it is also different from chain of thought: CoT enlarges test-time workspace by generating intermediate tokens, while prompt tuning and prefix tuning only change the conditioning interface to a frozen model.

A useful taxonomy is therefore: ICL-as-implicit-learning asks what algorithm the forward pass executes, prompt universality asks how large the prompt-induced function class is, and prompt limitations ask which invariants of the frozen backbone cannot be broken by any prompt.

### 6.1 In-Context Learning as Implicit Learning

This family treats the prompt as a training set and asks whether the transformer hidden states encode a current hypothesis that gets updated inside the forward pass.

**Representative result.** [Akyurek et al. 2023](https://openreview.net/forum?id=0g0X4H8yN4I) prove by construction that transformers can implement learning algorithms for linear models, including gradient-descent-style updates and closed-form regression. In their linear-regression setting, the prompt examples play the role of a dataset, and the model activations play the role of a temporary parameter vector or sufficient-statistic register.

**Proof sketch.** The construction allocates coordinates of the residual stream to quantities such as $X^\top X$, $X^\top y$, or the current iterate $w_t$. Attention aggregates information across the support examples, and a subsequent linear or nonlinear block carries out the update map. The key point is that the transformer does not merely imitate the input-output graph of regression; it carries out the algorithmic state update internally.

**Other major results.** [von Oswald et al. 2023](https://openreview.net/forum?id=tHvXrFQma5) give an explicit equivalence between one linear self-attention layer and one gradient-descent step. [Mahankali, Hashimoto, and Ma 2024](https://openreview.net/forum?id=8p3fu56lKc) show that the global optimum of a one-layer linear self-attention transformer trained on Gaussian linear regression implements one step of gradient descent, or preconditioned gradient descent under anisotropic covariates. [Fu et al. 2023](https://openreview.net/forum?id=dE5MEi9906) argue that trained transformers can realize higher-order optimization such as iterative Newton's method. The broader lesson is that ICL can often be read as a learned optimizer, not just as a pattern matcher.

### 6.2 Prompt Universality

Here the question is not which algorithm is executed, but how large the function class is when the backbone is frozen and only the prompt is allowed to change.

**Representative result.** [Petrov, Torr, and Bibi 2024](https://proceedings.mlr.press/v235/petrov24a.html) show that prefix tuning a single attention head is sufficient to approximate any continuous function, and that a transformer of depth linear in the sequence length can approximate arbitrary sequence-to-sequence functions. In their setting, a learned prefix can make a fixed backbone behave like a universal approximator.

**Proof sketch.** The proof is constructive: the prefix is chosen so that the fixed attention stack acts as a programmable interpolator or router, and approximation-theory arguments then control the error as a function of prefix length. The single-head result shows that attention itself can serve as the universal interface, while the sequence-length-dependent construction stacks local approximations to cover general seq2seq maps. The extra Jackson-type bounds quantify how much prefix length is needed for a target accuracy.

**Other major results.** [Wang, Chauhan, Wei, and Hsieh 2023](https://openreview.net/forum?id=zWxKYyW9ik) prove universality for soft-prompt tuning on a strong enough frozen transformer and, in the same paper, give lower bounds showing that finite-depth fixed-weight models still impose hard limits. The common theme is that a prompt can be a universal control surface only if the frozen backbone already has the right internal primitives.

### 6.3 Prompt Limitations

Universality is conditional. A frozen backbone can expose a large prompt space and still preserve invariants that prompts cannot change.

**Representative result.** [Petrov, Torr, and Bibi 2023](https://openreview.net/forum?id=GYOXIRXI7W) show that prompting and prefix tuning are strictly less expressive than full fine-tuning: context-based fine-tuning cannot change the relative attention pattern over content tokens and can only bias the output of an attention layer in a fixed direction. This means that prompts can elicit or combine skills already present in the backbone, but they cannot create a new attention pattern when the task requires one.

**Proof sketch.** The proof isolates structural quantities that remain invariant when only the prefix is changed. Since the content-token attention matrix is still computed by the frozen weights, prompt vectors can shift scores and outputs but cannot freely rewrite the relative ordering among content tokens. Tasks whose solution requires a different dependency graph or a different content-content attention pattern therefore remain out of reach.

**Other major results.** [Wang et al. 2023](https://openreview.net/forum?id=zWxKYyW9ik) construct datasets that cannot be memorized by any prompt length for a single encoder layer, derive lower bounds on the number of tunable prompt parameters, and extend the analysis to multi-layer settings where only invertible-function tasks remain learnable under their assumptions. The practical moral is that "continuous prompt space" is not the same thing as arbitrary reprogramming power.

Taken together, these results separate three axes that are easy to conflate. ICL-as-implicit-learning is about algorithmic behavior inside the forward pass; prompt universality is about what a frozen backbone can express when steered by prompts; prompt limitations identify the invariants that survive all prompt choices. CoT belongs to a different axis still, because it enlarges inference-time workspace rather than the prompt interface itself.
